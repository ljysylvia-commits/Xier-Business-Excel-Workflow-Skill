# EXCEL_AGENT_PRINCIPLES —— Agent 处理 Excel / CSV 的通用原则

> 本文只写通用原则，不写任何业务指标、渠道、平台或私有管线细节。
> 若运行环境已有 spreadsheet / xlsx / document / HTML 生成能力，优先使用成熟能力完成读写和渲染；本 Skill 负责流程、契约、溯源、校验与交付记录。
> 安装后或首次处理 Excel Dashboard/HTML 前，先按 `docs/ENVIRONMENT_READINESS.md` 做能力检查。

## 1. 定位

本 Skill 不是 Excel 引擎。它约束 Agent 在处理表格任务时必须：

1. 先理解结构，再写转换逻辑。
2. 保留从输出回到源数据的追溯路径。
3. 把业务口径、Agent 推断和假设写入 plan / CALIBERS。
4. 用独立校验证明结果可信。
5. 把可复用经验沉淀为候选 template/example，而不是直接写进 Skill 本体。
6. 先记录工具能力，再决定生成路径；降级路径必须可解释、可校验。
7. 缺少关键能力时先停下给安装建议，不把本地脚本写文件等同于完整 Excel 验收。

## 2. 读取 Excel 时

- 不只看第一张 Sheet；先列出 Sheet 名、可见性、表头位置、合并单元格、隐藏行列、冻结窗格和备注说明 Sheet。
- 区分“展示标题”和“真实字段”。合并标题、分组标题、跨列表头通常不是数据字段，需要展开成维度。
- 日期、百分比、千分位、货币、空值、`-`、`N/A`、空字符串要先统一类型，再计算。
- 公式单元格要确认读取的是公式、缓存值还是重算后的值；关键结果不能只信缓存。
- 清洗输出应优先采用长格式：一行一条“时间 × 维度 × 指标 × 口径 × 值”的观测。
- 清洗输出中尽量保留溯源列，例如 source_sheet、source_row、source_col、source_cell 或 source_range。

## 3. 写 Excel Dashboard 时

- 先定义受众和汇报问题，再定义布局。不要把源表结构直接搬成看板。
- Dashboard 中每个展示数都要能映射回源数据查询或计算公式。
- 样式可以参考用户样例，但不要把样例文件当作唯一执行依赖；脚本或工具调用要能复现格式。
- 数值计算使用全精度，只有最终展示层做四舍五入。
- 同比、环比、占比、差值、百分点差必须分别定义单位和分母。
- 空值、无基准、负值、异常值要有明确展示规则。
- 若使用公式、图表、条件格式或数据条，交付前要打开或重算检查，避免只有文件结构正确但可视结果错误。

## 4. 写 HTML 分析报告时

- HTML 报告面向业务阅读，不应只是数据表换皮。
- 必须包含：问题定义、数据范围、核心发现、指标拆解、对比或趋势、口径说明、行动建议。
- 所有关键数字要来自已清洗数据或可追溯计算，不能由模型凭上下文补数字。
- 图表和表格要服务结论；每个图表至少对应一个可验证的发现。
- 报告中的结论要区分事实、推断和建议。

## 5. 校验原则

- 清洗校验：输出每个值应能回到源单元格或源范围；无法逐格回溯时，要说明原因并设计替代校验。
- 分析校验：校验脚本不能 import 主脚本的计算函数，应从源数据独立重算关键结果。
- Dashboard / HTML 校验：抽取产物中的关键数，与独立重算结果逐项比对。
- 校验失败时不交付；修复后重跑，并把失败原因与修复记录写入 plan 第 8 章。

## 6. 与其他工具协作

- 有专门 spreadsheet / xlsx 能力时，优先让它负责 workbook 读写、样式、公式、图表和重算。
- 本 Skill 仍必须保存输入路径、输出路径、参数、关键口径、校验日志和产物清单。
- 不要把外部工具的成功退出等同于业务正确；业务正确必须由独立校验证明。

## 7. 工具链预检与降级

- 开始生成 Excel Dashboard / HTML 前，记录可用能力：spreadsheet/xlsx 工具、openpyxl、LibreOffice、浏览器/截图工具、HTML 静态检查能力。
- 不同环境的能力来源见 `docs/ENVIRONMENT_READINESS.md`；Codex / Claude Code / Antigravity 都按能力矩阵判断，不按名称硬编码。
- 首选成熟工具；若不可用，允许使用本地库或脚本降级生成，但必须在日志、`info.json` 和对应人读 info 视图中写明原因。
- 不能做截图或视觉渲染时，至少做替代验收：Sheet/DOM 结构、关键单元格/文本锚点、图表对象数量、行列尺寸、公式错误字符串、HTML 必要章节。
- 降级不等于降低业务校验标准；关键数字仍必须独立复算。

## 8. 合成平台风格数据声明

用于 eval、example 或 template 候选的“平台风格”数据必须写清：

- 数据为合成或已脱敏。
- “平台风格”只表示业务场景方向，不等于真实平台导出。
- 字段不等于任何平台私有字段。
- 报告中的建议仅服务测试或示例，不代表真实经营结论。
